# TvGU DataHub

TvGU DataHub — это модуль для сбора, нормализации, агрегации и структурирования данных Тверского государственного
университета (ТвГУ).

Проект предназначен для получения **целостной, связанной и пригодной для использования структуры данных** о:

- Преподавателях
- Дисциплинах
- Группах
- Факультетах / институтах
- Кафедрах (в разработке)
- Расписаниях занятий

Модуль используется как одна из баз данных проекта **BestTvGU**, но распространяется отдельно и открыт для повторного
использования.

---

## Основные возможности

- Асинхронный сбор данных из различных публичных источников ТвГУ
- Объединение расписаний всех групп
- Нормализация занятий (объединение одинаковых пар)
- Сопоставление преподавателей из расписаний с реальными профилями
- Генерация уникальных идентификаторов для всех сущностей
- Формирование единого агрегированного датасета

На выходе получается **единый JSON**, в котором все сущности связаны между собой по идентификаторам.

## Что собирается

### 📘 Структуры _(факультеты/институты)_

- Название

- Входящие группы
- Кафедры _(если доступны)_

- Контакты
- Преподаваемые дисциплины
- Группы и факультеты, где ведутся занятия

- Название

- День недели
- Номер пары
- Тип недели
- Предмет
- Преподаватели
- Группы
- Место проведения

## Нормализация данных

### Объединение занятий

Занятия, которые фактически являются одной парой, но встречаются в расписаниях разных групп, объединяются в одну
сущность.

При этом:

- Группы объединяются
- Преподаватели нормализуются
- Дубликаты устраняются

### Сопоставление преподавателей

Если в расписании указан преподаватель только по инициалам, DataHub пытается сопоставить его с реальным преподавателем
ТвГУ.

Используются:

- Совпадение инициалов
- Эвристическая оценка на основе:
    - Преподаваемых дисциплин
    - Образовательных направлений
    - Стажа работы

Поведение можно настраивать через конфигурацию.

## Конфигурация

В `config.py` доступны параметры:

```python
USE_HEURISTICS_FOR_TEACHERS = True
SKIP_UNRECOGNIZED_TEACHERS = False
```

`USE_HEURISTICS_FOR_TEACHERS` — использовать ли эвристики при совпадении нескольких кандидатов

`SKIP_UNRECOGNIZED_TEACHERS` — пропускать ли преподавателей, которых нельзя сопоставить

## Использование

### Как библиотека

```python
from tvgu_data_hub import get_all_tvgu_data
import asyncio

data = asyncio.run(get_all_tvgu_data())
```

Возвращаемая структура:

```json
{
    "departments": [...],
    "structs": [...],
    "teachers": [...],
    "places": [...],
    "subjects": [...],
    "groups": [...],
    "lessons": [...]
}
```

Все сущности имеют уникальные идентификаторы и связаны между собой.

_*Уникальные идентификаторы могут быть разными при различных запусках_

### Как CLI-инструмент

Экспорт всех данных в JSON:

```bash
python -m tvgu_data_hub -o all_tvgu_data.json --prettify
```

Или с автоматическим именем файла:

```bash
python -m tvgu_data_hub -oa -p
```

## Назначение проекта

TvGU DataHub создавался как открытый инфраструктурный слой:

- Для студенческих сервисов
- Для аналитики
- Для визуализации структуры университета
- Для интеграции с другими проектами

_Проект не связан с администрацией ТвГУ и использует только публично доступные данные_

## Связанные проекты

**BestTvGU** — основной пользовательский сервис на базе этих данных

Парсеры расписаний, преподавателей и структур ТвГУ _(используются как источники)_
